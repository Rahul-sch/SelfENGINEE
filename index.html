üî¨ How SABS Works

Security-Aware Beam Search integrates static analysis directly into the decoding loop:

  • Token-by-token verification - Checks security at syntactic boundaries
  • AST-based analysis - Detects shell execution, eval/exec, unsafe file ops
  • Adaptive penalties - Critical issues kill beams, warnings reduce score
  • Cache-optimized - LRU cache prevents redundant analysis
  • Configurable trade-offs - Adjust Œª to balance security vs. quality
ALGORITHM score = log_probability / length_penalty ‚àí Œª √ó security_penalty if CRITICAL issue: terminate beam else if WARNING: reduce score, continue else: proceed normally
